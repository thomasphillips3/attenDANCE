# Plan 04-04: Parent Invoice Payment (Stripe Elements) + Phase Verification

## Goal
Parents can view outstanding invoices and pay them online via embedded Stripe Elements, completing the parent portal and the full Phase 4 communications + portal feature set.

## Requirements
- PORT-02: Parents can view and pay outstanding invoices via Stripe

## Dependencies
- Plan 04-03 complete (parent portal, parent auth, parent routes)
- Plan 04-01 complete (payment receipt email triggers on webhook)
- Billing infrastructure from Phase 3 (invoices, Stripe SDK, webhook handler)

## Wave: 2

## Tasks

### T-04-04-01: Parent invoice API + Stripe PaymentIntent
**Files:**
- `server/src/routes/parent.ts` — add invoice routes
- `server/src/routes/billing.ts` — add create-payment-intent helper

**Details:**
- GET /parent/invoices — returns invoices for the parent's family (scoped by family_id from JWT), includes status, amount, due_date
- POST /parent/invoices/:id/pay — creates a Stripe PaymentIntent for the invoice amount
  - Ensures the invoice belongs to the parent's family
  - Uses family's stripe_customer_id (creates one if needed via ensureStripeCustomer)
  - Returns clientSecret for Stripe Elements frontend
  - Sets invoice metadata on the PaymentIntent for webhook reconciliation
- The existing webhook handler (invoice.payment_succeeded) already updates invoice status and sends receipt email

### T-04-04-02: Stripe Elements payment UI
**Files:**
- `client/package.json` — add `@stripe/stripe-js` and `@stripe/react-stripe-js`
- `client/src/screens/parent/ParentInvoices.tsx` — new screen, invoice list + payment
- `client/src/components/parent/PaymentForm.tsx` — Stripe Elements embed
- `client/src/hooks/useParent.ts` — add invoice queries + pay mutation
- `client/src/router.tsx` — add /parent/invoices route
- `client/src/layouts/ParentLayout.tsx` — add Invoices nav link

**Details:**
- `npm install @stripe/stripe-js @stripe/react-stripe-js` in client/
- ParentInvoices: list of invoices with status badges (pending=yellow, paid=green, overdue=red, waived=gray)
- Each unpaid invoice has a "Pay Now" button
- Clicking "Pay Now" calls POST /parent/invoices/:id/pay to get clientSecret, then opens PaymentForm modal
- PaymentForm: Stripe CardElement + submit button, handles payment confirmation
- On success: show confirmation message, refetch invoices (status updated via webhook)
- On error: show error message from Stripe
- Stripe publishable key from env var (VITE_STRIPE_PUBLISHABLE_KEY)
- All screens: mobile-first, 18px+ body text, 56px+ tap targets, studio branding

### T-04-04-03: Integration verification + cleanup
**Files:**
- No new files — verification of full Phase 4 integration

**Details:**
- Verify end-to-end flows:
  1. Admin enrolls student -> enrollment confirmation email sent -> logged in notification_log
