# Plan 04-02: SMS Notifications (Twilio) + Admin Broadcast

## Goal
The system sends SMS absence alerts and payment reminders via Twilio, admin can broadcast messages to families by class filter, and automated reminders fire for tuition due dates.

## Requirements
- COMM-04: SMS absence alerts to parents via Twilio
- COMM-05: SMS payment reminders via Twilio
- COMM-06: Admin broadcast email or SMS to families filtered by class
- COMM-07: Automated reminders for tuition due, missed payment, upcoming events

## Dependencies
- Plan 04-01 complete (notification service, notification_log table, Resend email integration)

## Wave: 2

## Tasks

### T-04-02-01: Twilio SDK setup + SMS service
**Files:**
- `server/package.json` — add `twilio` dependency
- `server/src/lib/notifications.ts` — add `sendSMS(opts)` function
- `server/src/types/index.ts` — add broadcast TypeBox schemas

**Details:**
- `npm install twilio` in server/
- Add `sendSMS(to, body, opts)` to notification service alongside existing `sendEmail`
- Lazy-init Twilio client (same pattern: error only when TWILIO_ACCOUNT_SID/AUTH_TOKEN/PHONE_NUMBER are missing and SMS is actually attempted)
- SMS sends log to notification_log with channel='sms'
- Fire-and-forget with error catching, same as email
- Note in code: A2P 10DLC registration required before production SMS (brand + campaign approval, 1-2 weeks)

### T-04-02-02: SMS triggers + automated reminders
**Files:**
- `server/src/routes/attendance.ts` — wire SMS absence alert alongside email
- `server/src/routes/billing.ts` — add payment reminder endpoint
- `server/src/lib/notifications.ts` — add reminder helpers

**Details:**
- Absence SMS trigger: on POST /sessions/:id/submit, for each absent student, also send SMS to family phone if phone exists
- Payment reminder: POST /billing/send-reminders (admin-only) — queries overdue invoices, sends SMS to each family with outstanding balance
- Automated reminder helper: `sendPaymentReminder(familyId, invoiceAmount, dueDate)` — "LSODance: Your tuition payment of $X is due on [date]. Pay online at [portal link]"
- SMS templates are plain text (no HTML), max 160 chars where possible
- Absence SMS: "LSODance: [Student] was marked absent from [class] on [date]. Contact us with questions."
- Payment SMS: "LSODance: Your payment of $[amount] is due [date]. Pay online: [link]"

### T-04-02-03: Admin broadcast endpoint + UI
**Files:**
- `server/src/routes/notifications.ts` — add POST /notifications/broadcast
- `client/src/screens/admin/NotificationsPage.tsx` — add broadcast compose form
- `client/src/hooks/useNotifications.ts` — add broadcast mutation

**Details:**
- POST /notifications/broadcast: admin-only, body: { channel: 'email'|'sms'|'both', classIds?: string[], subject?: string, message: string }
- If classIds provided, query enrolled students -> families -> deduplicate by family
- If no classIds, send to all active families in org
- For email: use a generic announcement template with studio branding
- For SMS: send plain text message
- Log each send to notification_log
- Client: broadcast compose form with channel selector (Email/SMS/Both), optional class filter multi-select, subject (email only), message textarea, send button with confirm modal
- Show send progress/result count after broadcast completes
