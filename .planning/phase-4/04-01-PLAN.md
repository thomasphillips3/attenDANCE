# Plan 04-01: Notification Infrastructure + Email (Resend)

## Goal
The system sends transactional emails for enrollment confirmations, payment receipts, and absence alerts — with every notification logged for audit and delivery tracking.

## Requirements
- COMM-01: Enrollment confirmation emails via Resend
- COMM-02: Payment receipt emails via Resend
- COMM-03: Absence notification emails to parents via Resend
- COMM-08: Notification log with delivery status

## Dependencies
- Phase 3 complete (billing routes, webhook handler, invoices, payments exist)
- Families table has email addresses (guardian_email from Phase 2)
- Attendance submission flow exists (Phase 1)

## Wave: 1

## Tasks

### T-04-01-01: Notification log migration + Resend SDK setup
**Files:**
- `supabase/migrations/20260522000800_notification_log.sql` — notification_log table
- `server/package.json` — add `resend` dependency
- `server/src/lib/notifications.ts` — new file, notification service module
- `server/src/types/index.ts` — add notification TypeBox schemas

**Details:**
- notification_log table: id, organization_id, family_id, channel (email/sms), recipient, subject, template_key, payload (jsonb), status (pending/sent/delivered/failed/bounced), external_id, error_message, created_at, sent_at
- Use existing enum types: notification_channel, delivery_status (already defined in migration 000100)
- RLS policy: admin-only read, system insert (service role)
- Notification service: `sendEmail(opts)` wrapper around Resend with logging to notification_log
- Lazy-init Resend client (same pattern as Stripe: error only when actually used, not on import)
- `npm install resend` in server/

### T-04-01-02: Email templates + trigger wiring
**Files:**
- `server/src/lib/email-templates.ts` — new file, HTML email template functions
- `server/src/routes/enrollments.ts` — wire enrollment confirmation email
- `server/src/routes/billing.ts` — wire payment receipt email
- `server/src/routes/attendance.ts` — wire absence notification email
- `server/src/routes/webhooks.ts` — wire payment receipt on invoice.payment_succeeded

**Details:**
- Plain HTML email templates (no React Email dependency — keep it simple):
  - `enrollmentConfirmation(studentName, className, studioName)` — "Welcome to [class]!"
  - `paymentReceipt(familyName, amount, invoiceDate, paymentMethod)` — "Payment received"
  - `absenceAlert(studentName, className, date)` — "[Student] was marked absent from [class]"
- All emails use studio branding: purple header bar, DM Serif Display heading, Atkinson Hyperlegible body
- Enrollment trigger: after successful `enroll_student()` RPC, look up family email, send confirmation
- Payment trigger: in webhook handler on `invoice.payment_succeeded`, send receipt to family email
- Absence trigger: on POST /sessions/:id/submit, for each student marked absent, send absence email to family
- Each send logs to notification_log via the notification service
- Fire-and-forget: don't await email delivery, catch errors and log them
