# Plan 04-03: Parent Portal — Auth + Family Views

## Goal
Parents can log in via magic link, view their enrolled students and class schedules, see attendance history, and update family contact information.

## Requirements
- PORT-01: Parents can log in and view enrolled students and classes
- PORT-03: Parents can update family contact information
- PORT-04: Parents can view attendance history for their students

## Dependencies
- Phase 2 complete (families, students, classes, enrollments exist)
- Phase 1 auth system (Supabase Auth, custom access token hook)

## Wave: 1

## Tasks

### T-04-03-01: Parent auth — role + magic link + access token hook
**Files:**
- `supabase/migrations/20260522000900_parent_auth.sql` — parent role support
- `server/src/routes/auth.ts` — add parent invite endpoint
- `server/src/plugins/auth.ts` — allow parent role through auth middleware

**Details:**
- Parents are NOT staff. Use families table for parent auth:
  - Add `parent_user_id uuid REFERENCES auth.users(id)` column to families table
  - Update custom_access_token_hook to check families table too: if user_id matches a family's parent_user_id, inject role='parent' + organization_id + family_id into app_metadata
  - This way parents get scoped to their family automatically via JWT claims
- Admin endpoint: POST /auth/invite-parent { familyId, email } — creates Supabase Auth user, links parent_user_id to family, sends magic link via supabase.auth.admin.inviteUserByEmail or signInWithOtp
- Auth middleware: allow 'parent' role to pass (parent routes scope to family_id from JWT)
- Magic link: Supabase handles the email delivery for OTP/magic link

### T-04-03-02: Parent-scoped API routes
**Files:**
- `server/src/routes/parent.ts` — new file, parent-facing routes
- `server/src/index.ts` — register parent routes

**Details:**
- All routes require role='parent', scope to family_id from JWT app_metadata
- GET /parent/dashboard — returns family info, enrolled students with class details
- GET /parent/classes — returns all classes the family's students are enrolled in, with schedule info
- GET /parent/attendance — returns attendance records for all family's students, supports ?studentId filter and date range
- GET /parent/profile — returns family record (guardians, email, phone, address)
- PATCH /parent/profile — update family contact info (guardian names, email, phone, emergency contact)
- Parent cannot modify students, enrollments, or billing — read-only except profile updates
