    - Admin role gate and organizationId check on every handler

    - POST /enrollments (enroll a student):
      Validate body against EnrollBody schema.
      Call the enroll_student Postgres function via RPC: const { data, error } = await fastify.supabase.rpc('enroll_student', { p_organization_id: organizationId, p_student_id: body.studentId, p_class_id: body.classId }).
      If error (Supabase/Postgres error), log and return 500 { error: 'Failed to enroll student' }.
      The RPC returns a single jsonb value. If data contains an 'error' key (typeof data === 'object' && data.error), return 409 { error: data.error }. This handles 'Class not found', 'Student already enrolled', 'Student already on waitlist'.
      Otherwise return 201 with the data object (enrollmentId, status, activeCount, capacity).

    - DELETE /enrollments/:id (drop a student):
      First verify the enrollment exists, belongs to this org, and is not already dropped: fastify.supabase.from('enrollments').select('id, status, organization_id').eq('id', params.id).maybeSingle(). If not found or org mismatch, return 404 { error: 'Enrollment not found' }. If status already 'dropped', return 409 { error: 'Student already dropped' }.
      Update: fastify.supabase.from('enrollments').update({ status: 'dropped', dropped_at: new Date().toISOString(), updated_at: new Date().toISOString() }).eq('id', params.id).eq('organization_id', organizationId).
      The promote_from_waitlist trigger fires automatically on this UPDATE.
      Return 200 { message: 'Student dropped', enrollmentId: params.id }.

    - POST /enrollments/transfer (transfer between classes):
      Validate body against TransferBody schema.
      Call transfer_student via RPC: fastify.supabase.rpc('transfer_student', { p_organization_id: organizationId, p_student_id: body.studentId, p_from_class_id: body.fromClassId, p_to_class_id: body.toClassId }).
      If Supabase error, log and return 500.
      If data.error, return 409 { error: data.error }.
      Return 200 with the data object (fromClassId, fromStatus, toClassId, toStatus, enrollmentId).

    Register enrollmentsRoutes in server/src/index.ts: import from './routes/enrollments.js', await fastify.register(enrollmentsRoutes).

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Elevation of Privilege | Enrollment endpoints | mitigate | request.role !== 'admin' check on every handler. enroll_student() verifies class belongs to org via WHERE clause. |
| T-02-14 | Tampering | POST /enrollments | mitigate | enroll_student() uses FOR UPDATE row lock on classes. Concurrent enrollments cannot exceed capacity. |
| T-02-15 | Tampering | DELETE /enrollments/:id | mitigate | Verify enrollment belongs to org before dropping. promote_from_waitlist trigger uses SKIP LOCKED to prevent double-promote on concurrent drops. |
| T-02-16 | Tampering | POST /enrollments/transfer | mitigate | transfer_student() is atomic (single Postgres transaction) per user decision. If target enroll fails, source drop is also rolled back. |
| T-02-SC | Tampering | No new npm packages | accept | No new packages installed in this plan. All dependencies from earlier plans. |
