SQL for GET /sessions/today (parameterized, $1 = organizationId):
  SELECT
    cs.id, cs.class_id AS "classId", cs.session_date AS "sessionDate", cs.status,
    c.name AS "className", c.start_time AS "startTime", c.duration_minutes AS "durationMinutes",
    (s.first_name || ' ' || s.last_name) AS "instructorName",
    COUNT(ar.id) FILTER (WHERE ar.status IN ('present','late')) AS "presentCount",
    COUNT(e.id) FILTER (WHERE e.status = 'active') AS "totalEnrolled"
  FROM class_sessions cs
  JOIN classes c ON c.id = cs.class_id AND c.organization_id = $1
  LEFT JOIN staff s ON s.id = c.instructor_id
  LEFT JOIN enrollments e ON e.class_id = cs.class_id AND e.organization_id = $1
  LEFT JOIN attendance_records ar ON ar.class_session_id = cs.id AND ar.organization_id = $1
  WHERE cs.organization_id = $1 AND cs.session_date = CURRENT_DATE
  GROUP BY cs.id, c.id, s.id
  ORDER BY c.start_time ASC

SQL for GET /sessions/:id/roster ($1 = organizationId, $2 = sessionId):
  SELECT
    e.id AS "enrollmentId", e.student_id AS "studentId",
    st.first_name AS "firstName", st.last_name AS "lastName",
    ar.id AS "attendanceId", ar.status AS "attendanceStatus"
  FROM enrollments e
  JOIN students st ON st.id = e.student_id AND st.organization_id = $1
  LEFT JOIN attendance_records ar
    ON ar.student_id = e.student_id
    AND ar.class_session_id = $2
    AND ar.organization_id = $1
  WHERE e.organization_id = $1
    AND e.class_id = (
      SELECT class_id FROM class_sessions WHERE id = $2 AND organization_id = $1
    )
    AND e.status = 'active'
  ORDER BY st.last_name ASC, st.first_name ASC

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> GET /sessions/today | JWT required; organizationId extracted from token only, never from query params |
| Browser -> GET /sessions/:id/roster | JWT required; session ownership verified against organizationId before data returned |
| IndexedDB cached_rosters -> UI | Client-side cache; stale data shown with offline indicator; refreshed on every successful fetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | GET /sessions/:id/roster IDOR | mitigate | Route verifies session.organization_id = request.organizationId from JWT before returning any data. Returns 403 on mismatch. |
| T-02-02 | Information Disclosure | Stale IndexedDB roster cache | accept | Cached data is scheduling info (student names, class times) — not financial or medical data. Cache refreshes on every successful network request. Risk is stale display, not data leak. |
| T-02-03 | Tampering | Client-side local attendance state | accept | localStatus in Plan 02 is UI-only optimistic display. No server writes occur in this plan. All writes validated server-side in Plan 03. |
| T-02-SC | Tampering | npm installs | mitigate | No new packages beyond those audited in 01-RESEARCH.md. |
