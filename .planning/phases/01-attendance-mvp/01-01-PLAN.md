From RESEARCH.md — Custom Access Token Hook (exact SQL pattern):
  Function: public.custom_access_token_hook(event jsonb) RETURNS jsonb LANGUAGE plpgsql STABLE
  Reads: SELECT role, organization_id FROM public.staff WHERE user_id = (event ->> 'user_id')::uuid
  Sets: claims := jsonb_set(claims, '{app_metadata,role}', to_jsonb(staff_record.role::text))
        claims := jsonb_set(claims, '{app_metadata,organization_id}', to_jsonb(staff_record.organization_id::text))
  Grant: GRANT EXECUTE ON FUNCTION public.custom_access_token_hook TO supabase_auth_admin
  Register: Supabase Dashboard > Authentication > Hooks > Custom Access Token Hook

From RESEARCH.md — RLS policy pattern (MUST use subselect — no bare auth.jwt() calls):
  ALTER TABLE {table} ENABLE ROW LEVEL SECURITY;
  ALTER TABLE {table} FORCE ROW LEVEL SECURITY;
  CREATE POLICY "tenant_isolation" ON {table} FOR SELECT
    USING (organization_id = (SELECT (auth.jwt() -> 'app_metadata' ->> 'organization_id')::uuid));

From RESEARCH.md — Fastify JWT preHandler pattern:
  const userClient = createClient(SUPABASE_URL, SUPABASE_ANON_KEY, { global: { headers: { Authorization: `Bearer ${token}` } } })

  Attach: request.organizationId = user.app_metadata.organization_id
  Attach: request.role = user.app_metadata.role
  Return 401 if !user or error
