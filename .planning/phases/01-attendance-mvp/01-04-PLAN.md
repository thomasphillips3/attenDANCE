POST /sessions/:id/submit contract:
  URL: /sessions/:id/submit
  Method: POST
  Auth: required (JWT preHandler — admin, front_desk, instructor roles permitted)
  Body: none
  Server behavior:
    1. SELECT organization_id FROM class_sessions WHERE id = params.id
    2. If no row or org mismatch with request.organizationId: return 403 { error: 'Forbidden' }
    3. UPDATE class_sessions SET status = 'completed', updated_at = now()
       WHERE id = params.id AND organization_id = request.organizationId
    4. Return 200 { sessionId: params.id, status: 'completed', submittedAt: new Date().toISOString() }
    5. On DB error: return 500 { error: 'Failed to submit attendance' }
  Note: class_sessions has no submitted_at column. submittedAt is computed at response time.
  Repeated calls to the same session are idempotent (UPDATE on already-completed is a no-op).

POST /rfid/checkin contract:
  URL: /rfid/checkin
  Method: POST
  Auth: required (any valid JWT)
  Body: { card_uid: string (required), device_id?: string }
  Behavior:
    1. If card_uid missing from body: return 400 { error: 'card_uid is required' }
    2. Return 501 { error: 'RFID check-in not yet implemented', message: 'Hardware integration pending Phase 2. card_uid received: ' + body.card_uid }
  Purpose: Reserves the URL and validates the request shape for ATTN-08. No DB writes.

POST /auth/invite contract:
  URL: /auth/invite
  Method: POST
  Auth: required AND request.role must equal 'admin'
  Body: { email: string, role: 'admin'|'instructor'|'front_desk' }
  Behavior:
    1. If request.role !== 'admin': return 403 { error: 'Admin role required' }
    2. Validate body (TypeBox schema)
    3. Call fastify.supabase.auth.admin.inviteUserByEmail(body.email, { data: { invited: true } })
    4. Return 200 { message: 'Invitation sent', email: body.email }
    5. On Supabase error: return 500 { error: error.message }

| Browser POST /auth/invite | JWT required AND role must be 'admin' — 403 for all other roles |

| Supabase auth.admin.inviteUserByEmail | Called with service role from Fastify only — never from the browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | POST /auth/invite role gate | mitigate | Route checks request.role === 'admin' before calling inviteUserByEmail. Returns 403 for front_desk and instructor. Admin role is in app_metadata (server-writable only via Auth Hook). |
| T-04-02 | Tampering | POST /sessions/:id/submit cross-org | mitigate | Route verifies session.organization_id = request.organizationId before UPDATE. Returns 403 on mismatch. |
| T-04-03 | Information Disclosure | RFID stub echoes card_uid in 501 response | accept | card_uid is a hardware UID containing no PII. Response returned only to authenticated callers. |
| T-04-04 | Denial of Service | Repeated submit on same session | accept | UPDATE class_sessions SET status = 'completed' on an already-completed session is a no-op. Idempotent by nature. |
| T-04-05 | Tampering | Password reset redirectTo | accept | redirectTo is set to window.location.origin (the app's own origin). Supabase validates redirect URLs against the allowlist configured in the project settings. |
| T-04-SC | Tampering | npm installs | mitigate | No new npm packages. @radix-ui/react-dialog was audited in 01-RESEARCH.md. |
</threat_model>
