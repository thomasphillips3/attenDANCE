# Plan 05-03: Staff/Instructor Portal — Schedule, Attendance, Hours

## Goal
Instructors can view their assigned class schedules, mark attendance for their own classes, and log hours worked.

## Requirements
- STAF-01: Instructor schedule view
- STAF-02: Instructor marks attendance for own classes
- STAF-03: Hour logging

## Dependencies
- Plan 05-01 (dashboard infrastructure)
- Existing: staff table with hourly_rate column, class_instructors table, attendance routes

## Tasks

### Task 1: Staff API endpoints
**File:** `server/src/routes/staff.ts` (new file, separate from GET /staff in classes.ts)

Create instructor-scoped endpoints:

1. `GET /staff/me/schedule` — Returns classes assigned to the logged-in instructor (via class_instructors or classes.instructor_id), including day_of_week, start_time, duration, room, enrolled count. Filtered to active assignments only.

2. `GET /staff/me/sessions` — Returns class_sessions for the instructor's classes, with query params: `date` (default today), `week` (returns full week). Includes attendance summary per session.

3. `POST /staff/hours` — Log hours: `{ date, hours, class_id?, notes? }`. Creates a row in a new `staff_hours` table.

4. `GET /staff/me/hours` — Returns logged hours for the current instructor, with query params: `start_date`, `end_date`. Includes calculated total and pay (hours * hourly_rate).

Auth: instructor or admin role. Instructor endpoints filter to own data only.

### Task 2: Staff hours migration
**File:** `supabase/migrations/20260522001000_staff_hours.sql`

Create staff_hours table with columns: id, organization_id, staff_id, class_id, date, hours (numeric 4,2), notes, created_at, updated_at.

Add RLS policies matching the org-scoped subselect pattern. Instructors can read/write their own hours; admins can read all.

### Task 3: Instructor portal UI
**Files:** `client/src/screens/staff/InstructorDashboard.tsx`, `client/src/screens/staff/InstructorSchedule.tsx`, `client/src/screens/staff/InstructorHours.tsx`, `client/src/layouts/InstructorLayout.tsx`

Build an instructor-facing section at `/instructor`:
- **InstructorLayout**: Simple layout with top nav (My Schedule | My Classes | Hours)
- **InstructorDashboard**: Today's classes with "Mark Attendance" button linking to the existing roster view
- **InstructorSchedule**: Weekly view of assigned classes
- **InstructorHours**: Log hours form + history table with date, hours, class, notes, and running total/pay calculation

Add routes in `router.tsx` under `/instructor` path.
