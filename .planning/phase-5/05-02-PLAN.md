# Plan 05-02: Reports — Enrollment, Revenue, Attendance + CSV Export

## Goal
Admin can generate enrollment, revenue, and attendance rate reports and export any of them as CSV.

## Requirements
- DASH-03: Enrollment report
- DASH-04: Revenue report
- DASH-05: Attendance rate report
- DASH-06: CSV export for all reports

## Dependencies
- Plan 05-01 (dashboard exists, Reports sidebar link is placeholder)

## Tasks

### Task 1: Report API endpoints
**File:** `server/src/routes/reports.ts`

Create 3 report endpoints, all admin-only:

1. `GET /reports/enrollment` — query params: `start_date`, `end_date`
   Returns: per-class breakdown of active/waitlist/dropped counts, total active students, new enrollments in period, drops in period.

2. `GET /reports/revenue` — query params: `start_date`, `end_date`
   Returns: total collected (sum of payments.amount), total outstanding (sum of pending invoices), total overdue (sum of overdue invoices), breakdown by payment method (stripe/cash/check), monthly trend array.

3. `GET /reports/attendance` — query params: `start_date`, `end_date`
   Returns: overall attendance rate (present+late / total records), per-class attendance rates, per-day-of-week rates, top absent students.

All responses include a `rows` array formatted for direct CSV conversion.

Register the plugin in `server/src/index.ts`.

### Task 2: Reports page with tabbed UI
**File:** `client/src/screens/admin/ReportsPage.tsx`

Build a reports page with:
- Tab bar: Enrollment | Revenue | Attendance
- Date range picker (start/end date inputs, default to current month)
- Summary cards at top of each tab showing key metrics
- Data table below with the detailed breakdown
- "Export CSV" button (purple, top-right) that converts the `rows` array to CSV and triggers browser download

Use TanStack Query with the date range as query key. Follow existing component styling patterns.

### Task 3: Wire Reports into sidebar + router
**Files:** `client/src/components/admin/AdminSidebar.tsx`, `client/src/router.tsx`

- Activate "Reports" from grayed `<span>` to `<NavLink to="/admin/reports">`
- Add lazy-loaded route for `/admin/reports` -> `ReportsPage`
