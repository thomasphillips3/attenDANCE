# Plan 05-01: Admin Dashboard — KPI Cards + Today's Classes

## Goal
Carollette sees a live operations dashboard when she opens the admin panel: 4 KPI cards (classes today, students checked in, absences, RFID check-ins) and today's class summary cards with attendance counts and status.

## Requirements
- DASH-01: Daily KPI cards
- DASH-02: Today's class summary cards with attendance counts

## Dependencies
- Phase 4 complete (all data sources exist)
- Existing routes: attendance, classes, class_sessions, attendance_records tables
- AdminSidebar has grayed "Dashboard" link — activate it

## Tasks

### Task 1: Dashboard API endpoint
**File:** `server/src/routes/dashboard.ts`

Create `GET /dashboard/today` endpoint that returns:
- `classes_today`: count of class_sessions where session_date = today
- `students_checked_in`: count of distinct attendance_records with status IN ('present','late') for today's sessions
- `total_enrolled_today`: count of distinct enrollments for today's classes
- `absences_today`: count of attendance_records with status = 'absent' for today's sessions
- `excused_today`: count where status = 'excused'
- `rfid_checkins_week`: count of attendance_records where marked_by = 'rfid' in last 7 days
- `class_summaries`: array of today's classes with: class name, time, instructor name, present count, absent count, total enrolled, session status, marked_by (staff name or 'RFID')

Register the plugin in `server/src/index.ts`.

Auth: admin or front_desk role required.

### Task 2: Dashboard page component
**File:** `client/src/screens/admin/DashboardPage.tsx`

Build the dashboard page matching the `admin.jsx` mockup design:
- Top bar: date display + "Today at the studio" headline
- 4 KPI cards in a grid row: Classes today (purple), Students checked in (green), Absences today (red), RFID check-ins (blue/indigo)
- Each KPI card: uppercase label, large number (DM Serif Display), subtitle
- Today's classes: grid of SummaryCard components (7 columns on desktop, responsive)
- Each SummaryCard: time, class name, instructor, present/total count, absent count, progress bar, status badge (Complete/In progress/Pending)

Use TanStack Query for data fetching. Follow existing component patterns.

### Task 3: Activate Dashboard in sidebar + router
**Files:** `client/src/components/admin/AdminSidebar.tsx`, `client/src/router.tsx`

- Change Dashboard from grayed `<span>` to active `<NavLink to="/admin/dashboard">`
- Add lazy-loaded route for `/admin/dashboard` -> `DashboardPage`
- Change admin index redirect from "students" to "dashboard"
- Also activate "Attendance" link -> redirect to front desk attendance view (`/`)
- Activate "Reports" link -> placeholder (will be built in Plan 05-02)
