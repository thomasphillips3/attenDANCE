# Plan 03-01: Stripe SDK + Tuition Plan & Discount CRUD

## Goal
Admin can create and manage tuition plans per class and apply discounts to families — the pricing foundation that every other billing plan depends on.

## Requirements
- BILL-01: Tuition plans per class (monthly, per-session, seasonal)
- BILL-04: Sibling, scholarship, and staff discounts

## Dependencies
- Phase 2 complete (classes + families + enrollments exist)
- Billing schema already in place (migration 000100)
- RLS policies already in place (migration 000200)

## Wave: 1

## Tasks

### T-03-01-01: Install Stripe SDK and add billing types
**Files:**
- `server/package.json` — add `stripe` dependency
- `server/src/types/index.ts` — add TypeBox schemas for tuition plans and discounts

**Details:**
- `npm install stripe` in server/
- Add TypeBox schemas: CreateTuitionPlanBody, UpdateTuitionPlanBody, CreateDiscountBody, UpdateDiscountBody
- Tuition plan fields: classId, amount, interval (monthly/per_session/seasonal)
- Discount fields: familyId?, classId?, type (sibling/scholarship/staff), amount or percent

### T-03-01-02: Tuition plan + discount server routes
**Files:**
- `server/src/routes/billing.ts` — new file, tuition plan + discount CRUD
- `server/src/index.ts` — register billing routes

**Details:**
- Follow existing route pattern (fastify-plugin, admin role gate, organization scope)
- Tuition plans: GET /tuition-plans, GET /tuition-plans/:id, POST /tuition-plans, PATCH /tuition-plans/:id, DELETE /tuition-plans/:id
- Discounts: GET /discounts, POST /discounts, PATCH /discounts/:id, DELETE /discounts/:id
- GET /tuition-plans supports ?classId filter
- GET /discounts supports ?familyId and ?classId filters
- All mutations admin-only

### T-03-01-03: Client hooks and billing pages
**Files:**
- `client/src/hooks/useTuitionPlans.ts` — new hook, TanStack Query
- `client/src/hooks/useDiscounts.ts` — new hook, TanStack Query
- `client/src/screens/admin/BillingPage.tsx` — new screen, tuition plan list + discount panel
- `client/src/screens/admin/TuitionPlanForm.tsx` — new screen, create/edit tuition plan
- `client/src/components/admin/AdminSidebar.tsx` — add Billing nav item (active)
- `client/src/router.tsx` — add /admin/billing, /admin/billing/plans/new, /admin/billing/plans/:id routes

**Details:**
- Follow existing hook pattern (useClasses, useStudents): TanStack Query with queryKey arrays
- BillingPage shows tuition plans table + discounts section
- TuitionPlanForm: select class, enter amount, choose interval
- Discount management inline on BillingPage (add/remove discount for family/class)
- 56px+ tap targets, Atkinson Hyperlegible body text, existing design tokens
