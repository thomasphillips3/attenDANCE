# Plan 03-02: Invoice Generation + Manual Payments + Stripe Customer

## Goal
Admin can generate invoices for families based on their enrollments and tuition plans, record cash/check payments, and families get Stripe Customer IDs for future recurring charges.

## Requirements
- BILL-02: Auto-generate invoices from enrollment + tuition plans
- BILL-05: Payment history by family (paid, pending, overdue, waived)
- BILL-06: Manual cash/check payment recording

## Dependencies
- Plan 03-01 (tuition plans + discounts exist)

## Wave: 2

## Tasks

### T-03-02-01: Stripe customer creation + invoice generation routes
**Files:**
- `server/src/routes/billing.ts` — add invoice routes + Stripe customer helper
- `server/src/types/index.ts` — add invoice + payment TypeBox schemas

**Details:**
- Stripe customer auto-creation: when generating an invoice for a family with no stripe_customer_id, create a Stripe Customer and save the ID to families.stripe_customer_id
- POST /invoices/generate — admin triggers invoice generation for a family (looks up enrollments, tuition plans, calculates amount with discounts, creates invoice row)
- GET /invoices — list invoices, filterable by ?familyId, ?status
- GET /invoices/:id — invoice detail with line items breakdown
- PATCH /invoices/:id — update status (waive an invoice)
- Invoice amount calculation: sum tuition plan amounts for all active enrollments, subtract applicable discounts
- Initialize Stripe client with STRIPE_SECRET_KEY env var

### T-03-02-02: Manual payment recording routes
**Files:**
- `server/src/routes/billing.ts` — add payment routes

**Details:**
- POST /payments — record a manual payment (cash or check), links to an invoice
- Body: { invoiceId, amount, method: 'cash' | 'check', notes? }
- On successful payment that covers the invoice amount, update invoice status to 'paid'
- Partial payments allowed — invoice stays 'pending' until fully covered
- GET /payments?invoiceId= — list payments for an invoice
- Admin + front_desk roles can record payments

### T-03-02-03: Invoice + payment client UI
**Files:**
- `client/src/hooks/useInvoices.ts` — new hook
- `client/src/hooks/usePayments.ts` — new hook
- `client/src/screens/admin/InvoicesPage.tsx` — new screen, invoice list with status filters
- `client/src/components/admin/RecordPaymentModal.tsx` — new component, cash/check payment form
- `client/src/components/admin/GenerateInvoiceModal.tsx` — new component, select family then generate
- `client/src/router.tsx` — add /admin/billing/invoices route

**Details:**
- InvoicesPage: table with columns (Family, Amount, Status, Due Date, Actions)
- Status badges: pending (yellow), paid (green), overdue (red), waived (gray)
- "Record Payment" button opens modal with amount, method, notes fields
- "Generate Invoice" button opens modal to select family and generate
- Follow existing modal pattern (ConfirmModal, EnrollmentModal)
