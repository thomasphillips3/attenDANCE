# Plan 03-03: Stripe Webhook Handler + Recurring Billing

## Goal
Stripe charges recurring payments automatically on the billing date, and the webhook handler processes payment events idempotently — no duplicate charges, no missed state updates.

## Requirements
- BILL-03: Recurring Stripe charges on billing date
- BILL-07: Failed payment retry + past_due/unpaid tracking
- BILL-08: Idempotent Stripe webhook handler

## Dependencies
- Plan 03-02 (invoices + Stripe customer exist)

## Wave: 3

## Tasks

### T-03-03-01: Stripe webhook endpoint with idempotency
**Files:**
- `server/src/routes/webhooks.ts` — new file, POST /webhooks/stripe
- `server/src/index.ts` — register webhook route, configure raw body parsing

**Details:**
- POST /webhooks/stripe — receives Stripe webhook events
- Verify webhook signature using STRIPE_WEBHOOK_SECRET env var
- Idempotency: INSERT INTO processed_webhook_events ON CONFLICT DO NOTHING, check rows affected
- Must return 200 immediately — async processing via the webhook_events pattern
- Handle events: invoice.payment_succeeded, invoice.payment_failed, customer.subscription.created, customer.subscription.updated, customer.subscription.deleted, customer.subscription.trial_will_end
- Webhook route MUST skip auth middleware (Stripe calls it, not a logged-in user)
- Raw body required for signature verification — configure Fastify content type parser
- Use service role Supabase client (bypasses RLS) for webhook event processing

### T-03-03-02: Stripe subscription creation from tuition plans
**Files:**
- `server/src/routes/billing.ts` — add subscription management routes

**Details:**
- POST /billing/subscribe — create a Stripe Subscription for a family's enrollments
- Flow: look up family's Stripe Customer, create Stripe Price from tuition plan (if no stripe_price_id), create Subscription with those prices
- PATCH /billing/subscribe/:subscriptionId — cancel or pause a subscription
- Only 'monthly' interval tuition plans create Stripe Subscriptions
- 'per_session' and 'seasonal' generate one-time invoices (handled in Plan 03-02)
- Store stripe_price_id on tuition_plans table after creation

### T-03-03-03: Webhook event handlers
**Files:**
- `server/src/routes/webhooks.ts` — add handler functions for each event type

**Details:**
- `invoice.payment_succeeded`: find local invoice by stripe_invoice_id, insert payment record, set invoice status to 'paid'
- `invoice.payment_failed`: find local invoice, set status to 'overdue', log for retry
- `customer.subscription.deleted`: find enrollments for this family, mark invoice status appropriately
- Failed payment handling: Stripe's built-in Smart Retries handle retry logic (3 attempts over ~3 weeks). Our handler just tracks the state transitions (pending to overdue to unpaid if all retries fail)
- After final retry failure (invoice.payment_failed with next_payment_attempt = null), set invoice status to 'overdue' and flag for admin attention
